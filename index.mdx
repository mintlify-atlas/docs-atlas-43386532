---
title: "sema — Go Semaphore Library"
description: "A lightweight, zero-dependency Go library for controlling goroutine concurrency with a simple channel-based semaphore."
---

`sema` gives you a minimal, idiomatic Go semaphore to limit how many goroutines run at the same time. It wraps a buffered channel behind a clean interface so you can control parallelism in worker pools, API clients, file processors, and anywhere else you need to bound concurrent work.

## Get started

<Steps>
  <Step title="Install the package">
    Add `sema` to your Go module with a single command:

    ```bash
    go get -u github.com/andreimerlescu/sema
    ```
  </Step>
  <Step title="Create a semaphore">
    Call `sema.New` with the maximum number of goroutines you want running at once:

    ```go
    import "github.com/andreimerlescu/sema"

    sem := sema.New(10) // allow up to 10 concurrent goroutines
    ```
  </Step>
  <Step title="Acquire and release slots">
    Wrap your goroutine body with `Acquire` before and `Release` (via `defer`) after:

    ```go
    sem.Acquire()
    go func() {
        defer sem.Release()
        // your work here
    }()
    ```
  </Step>
</Steps>
